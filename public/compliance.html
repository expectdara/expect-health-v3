1. Platform Overview

Expect Health is a digitally-delivered pelvic floor physical therapy platform that provides validated clinical screening, automated care plan generation, and physical therapist oversight. The platform serves patients who lack access to specialty pelvic floor care, particularly in rural and underserved communities.

Key Design Principles
  • Deterministic clinical logic — All scoring, tier assignment, exercise selection, and ICD-10 diagnosis is rule-based. No large language model (LLM), generative AI, or machine learning is used at runtime for clinical decisions.
  • PT-in-the-loop — Every care plan is reviewed and approved by a licensed physical therapist before reaching the patient.
  • Auditable by design — Every clinically significant action generates a timestamped, immutable audit event.
  • Minimal data collection — The platform collects only the data necessary for clinical screening and care plan generation.

PHI Masking

The platform includes a built-in PHI masking system for auditor access. When Auditor Mode is enabled, 17 categories of PHI fields are automatically masked using one-way hashing, allowing clinical pattern review without exposing patient identity. Masked fields include: name, email, DOB, phone, fax, SSN, MRN, address, and clinical notes.

Data Not Collected

  • Social Security numbers (not required for clinical screening)
  • Financial or payment information (not processed on-platform)
  • Geolocation data (Permissions-Policy: geolocation=())
  • Camera/photo data (Permissions-Policy: camera=())

Session Management

  • Session timeout: 15 minutes of inactivity triggers warning; 60-second countdown to automatic logout
  • Activity detection: Click, keypress, touch, and scroll events reset the idle timer
  • Logout actions: Server-side session deletion, localStorage clearance, auth token nullification, audit event logged
  • Token storage: Session tokens stored in localStorage with server-side validation on each request; tokens are cryptographically generated UUIDs
  • Session expiration: 60-minute absolute expiration enforced server-side regardless of activity

VA OAuth 2.0 Integration

  • Authorization Code Grant with PKCE (Proof Key for Code Exchange)
  • Code verifier generated client-side using crypto.getRandomValues (48 bytes)
  • SHA-256 code challenge transmitted; code verifier never leaves the client until token exchange
  • Access tokens used only for immediate FHIR resource fetch, not persisted
  • Redirect URI strictly validated against VA-registered callback URL

Audit Record Structure

Each event includes: unique event ID, ISO 8601 timestamp, event type, associated user ID, and event-specific details. Events are append-only and cannot be modified or deleted through the application interface.

Care Plan Review Workflow

  1. Patient completes validated clinical intake (all instruments scored deterministically)
  2. Platform generates personalized care plan using rule-based clinical logic
  3. Care plan enters PT review queue with risk flags highlighted
  4. Licensed PT reviews plan, exercises, adjuncts, and diagnoses
  5. PT approves, modifies, or rejects the plan (all actions audit-logged)
  6. Only approved plans are delivered to the patient

Clinical Logic Versioning

All scoring algorithms, tier assignments, and care plan generation logic carry a version identifier (CLINICAL_LOGIC_VERSION). Any change to clinical logic increments the version, and the version is recorded in every outcome record for traceability.

9. HIPAA Compliance

Technical Safeguards
  • Encryption in transit: TLS 1.2+ enforced on all connections (HSTS with 1-year max-age)
  • Encryption at rest: Database provider (Convex) encrypts all stored data using AES-256
  • Access controls: Role-based access (Patient, PT, OAIP Auditor) with credential-based authentication
  • Audit controls: Comprehensive audit logging of all access and clinical actions
  • Automatic logoff: 15-minute idle timeout with 60-second warning
  • Integrity controls: Audit events are append-only; clinical logic is versioned
Administrative Safeguards
  • Designated Privacy Officer: Dara Cook, Founder & CEO
  • Workforce training: All team members with PHI access complete HIPAA training
  • Business Associate Agreements: BAAs in place with infrastructure providers handling PHI
  • Minimum necessary standard: Each role accesses only the PHI required for its function
  • OAIP auditor access: PHI masking enforced — auditors see de-identified data only
BAA Readiness

Expect Health is prepared to execute a Business Associate Agreement with any covered entity or health system partner. We maintain BAAs with all subprocessors who handle PHI on our behalf.

10. Incident Response

Breach Notification

  • Suspected breaches are investigated within 24 hours of discovery
  • Affected individuals notified within 30 days of confirmed breach (ahead of the HIPAA 60-day requirement)
  • HHS Office for Civil Rights notified as required by HIPAA Breach Notification Rule
  • Covered entity partners (health systems) notified within 24 hours of confirmed breach

Vulnerability Management

  • Dependencies monitored for known vulnerabilities via automated tooling
  • Critical patches applied within 72 hours of disclosure
  • Infrastructure providers (Vercel, Convex) maintain their own vulnerability management and patching programs
  • Application code reviewed for OWASP Top 10 vulnerabilities

11. Data Retention & Deletion

  • Active patient data: Retained for the duration of the care episode plus 7 years (consistent with medical record retention requirements)
  • Audit logs: Retained for the duration of the OAIP pilot plus 3 years for regulatory compliance
  • Session data: Automatically purged on logout or timeout; server-side sessions expire after 60 minutes
  • Draft intake data: Stored in browser sessionStorage; expires after 72 hours; cleared on completion or abandonment
  • VA access tokens: Used for immediate FHIR fetch only; not persisted to database or localStorage

Patient Data Deletion

Patients may request permanent deletion of all their data by contacting privacy@expect.care. Deletion is completed within 45 days of the request. Deletion includes all PHI, intake responses, care plans, adherence data, and outcome records. De-identified audit events may be retained for regulatory compliance.
